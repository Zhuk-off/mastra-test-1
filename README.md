# Чистильщик лендингов

AI-агент-«верстальщик» для арбитража трафика. Цель — закрыть **весь цикл** подготовки
лендинга под оффер:

```
задание → получить лендинг → очистить → адаптировать → проверить → залить в трекер → метрика
```

Сейчас реализовано **ядро**: скачать → очистить → проверить. Полная карта всех этапов
со статусом — в [ARCHITECTURE.md (§10)](ARCHITECTURE.md). Построен на
[Mastra](https://mastra.ai/) (TypeScript).

---

## Что делает каждый этап

1. **Задание** — что сделать с лендингом (тексты, картинки, оффер). Сейчас — через **чат**
   (скопировал задание и файлы → отдал агенту); интеграция с YouGile — опционально, на будущее.
2. **Получить лендинг** — скачать из интернета (Playwright) или забрать из трекера (Keitaro).
3. **Очистить** — убрать аналитику, трекеры и кражу трафика, привести к нормальной структуре.
   Принцип: **белый список + карантин** (неизвестное не удаляется молча, а изолируется для ревью).
4. **Адаптировать** — перевод текстов, замена картинок на оффер, наши базовые скрипты.
5. **Проверить** — что работает и корректно отображается (десктоп + мобайл).
6. **Залить** в трекер, проверить на проде, подключить метрику (Clarity).

Реализованы шаги 2–3 и проверка (шаг 5: десктоп + мобайл, визуальный diff с оригиналом).

## Быстрый старт

**Требования:** WSL (Ubuntu-24.04), Node `>=22.13.0` через nvm.

```bash
npm install                       # зависимости (+ playwright chromium)
cp .env.example .env              # вписать ключ модели

npm run download -- <url>              # скачать лендинг в downloads/<host>/
npm run clean    -- downloads/<host>   # очистить
npm run verify   -- downloads/<host>   # проверить, что не звонит наружу

npm run dev                       # Mastra Studio (UI агента) на localhost:4111
```

Запустить ядро можно двумя путями: командами выше (CLI) или через агента `landing-agent`
(он вызывает те же 3 инструмента) в Mastra Studio / чате.

Флаги очистки: `--no-backup`, `--no-advanced`, `--coverage`, `--coverage-threshold=<n>`.
После очистки в папке появляются `clean-report.md` (что сделано), `clean-site-changes.log`
(детальный лог) и `_quarantine/` (изолированное подозрительное — на ручное ревью).

## Структура проекта

| Папка | За что отвечает |
| --- | --- |
| `scripts/` | CLI: запуск конвейера руками |
| `src/mastra/` | весь код: агент, инструменты, ядро очистки |
| `src/mastra/cleaners/` | ❤️ ядро очистки (проходы, правила, утилиты, verify) |
| `docs/` | спеки очистки + аудит безопасности (red-team) |
| `downloads/` | рабочие данные: сюда качаются и тут чистятся лендинги |

Полная карта с разбором каждого файла — в [ARCHITECTURE.md](ARCHITECTURE.md).

## Разработка

```bash
npx vitest run        # тесты
npx tsc --noEmit      # проверка типов
```

При добавлении нового агента/инструмента/воркфлоу — регистрируй его в `src/mastra/index.ts`.
Новые правила очистки добавляются **только** через новые проходы в `src/mastra/cleaners/passes/`.
