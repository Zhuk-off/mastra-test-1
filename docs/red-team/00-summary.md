# Red Team: сводка и приоритеты (capstone)

Итог red-team разбора всей системы очистки. Полный реестр находок и навигация — в [`_index.md`](_index.md);
здесь — **корневые кластеры** (мало фиксов закрывают много находок) и **порядок работ**.

**Охват:** 6 тиров, ~30 исходников + реестры, **~130 пунктов** (находки + заметки + сильные стороны),
из них **4 Critical** и **~17 High**. Все сквозные нити T-1…T-9 закрыты.

> Это анализ, не правки. Цель — выбрать, что чинить, и не сломать то, что работает.

## 4 Critical (читать первыми)

| ID | Где | Суть |
|----|-----|------|
| [AL-1](allowlist.md) | allowlist.ts | `data:`/`javascript:`/`blob:` в `<script>`/`<iframe>`/`href` → `keep`. Исполняемый чужой код переживает очистку. Подтверждено сквозь (2A-2, 2D-1). |
| [PIPE-1](pipeline.md)+[DOM-1](html-dom.md) | pipeline / html-dom | Серверный тег (даже текст `<? ` или `<% %>`) → `hasServerTags` → **вся** HTML-очистка молча пропущена. Тривиальный обход всего. Замер прилагается. |
| [NORM-1](normalize-landing-structure.md) | normalize | Path traversal: `<img src="../../../.aws/credentials">` → `moveFileUnique` `link`+`unlink` → перемещает/удаляет файл вне siteDir и кладёт в собранный сайт (destruction + exfil). |

(DOM-1 и PIPE-1 — один баг с двух сторон, считаю как один Critical-кластер.)

## Корневые кластеры (фикс → множество находок)

### C1 · Нормализация URL и схемы в `classifyResource` 🔴 P0
**Один фикс закрывает целый класс обхода.** В `allowlist.ts`: (а) `url.trim()` + удалить `\t\r\n`
до классификации; (б) классифицировать не-http схемы по типу ресурса (`data:`/`blob:`/`javascript:`/
`vbscript:` для script/iframe/media → quarantine; `javascript:` в href → remove; `data:` для img — ок).
**Закрывает:** AL-1, AL-2, 2A-1, 2A-2, 2D-1, частично CSS-4/SVG-`//`. Дёшево, эффект максимальный,
подтверждено эмпирикой.

### C2 · Убрать «тихий SKIP всей очистки» 🔴 P0
Серверные блоки — не повод выключать всё. Варианты: защитить-и-восстановить (плейсхолдер→парс→чистка→
возврат) **или** карантинить файл и **громко** падать в статусе. Сузить `hasServerTags` (нет FP на
`<%`/текст `<? `). **Закрывает:** PIPE-1, DOM-1, NORM-2, 2D-4 (иначе на этих файлах нет даже CSP).

### C3 · Sandbox-границы файловой системы 🔴 P0 / 🟧 P1
(а) В `normalize` после `resolve` проверять, что путь **внутри** siteDir (`relative` без ведущего `..`)
— закрывает **NORM-1** (Critical). (б) Трактовать `_external/<host>/` как внешний ресурс этого хоста и
гнать через `classifyResource` — закрывает **EXT-1** (чужой контент перестаёт быть «локальным»).

### C4 · Детекторы: не только литерал + прямой вызов 🟧 P1
Вся JS/PHP-детекция обходится косвенностью. Минимум: резолвить простые алиасы/строковые константы;
матчить member/bracket-формы (`window.fetch`, `el['onkeydown']`, `var img=new Image();img.src=`);
**нелитеральный** сетевой URL — флагать как подозрительный; единый `isExternalUrl` с обработкой `//`.
**Закрывает (FN):** DET-1/2/3, DEC-1, EVAL-1, SW-2, PARSE-1, RED-1, KEY-1, DOC-1, PHP-1.

### C5 · Destructive-delete → карантин + видимость 🟧 P1
HTML-слой правильно **карантинит**; JS/coverage — **удаляют файлы целиком** по эвристике (FP-дорого,
необратимо), а отчёт это **скрывает**. Фиксы: (а) [REP-1](5a-utils-infra.md) — починить тип
`PHP_BACKDOOR_WARN` и **перечислять удалённые файлы** в отчёте (дёшево, P0-кандидат); (б) обфускация/
metric → карантин, не `unlink` (CJS-2/OBF-1/MET-1); (в) AST-хирургия: перепарсить после мутации /
удалять охватывающий statement / стаб вместо удаления функции (**CJS-1**, **EUF-1**, RIE-1, DEC-1).

### C6 · Не использовать блок-лист в обход белого списка 🟨 P2
Места, где allowlist НЕ применяется и работает голый блок-лист → FN: preconnect/preload (2A-3),
`<noscript>` (2A-4), inline-`<script>` (2A-5), CSS `url()` + inline-CSS (CSS-1/CSS-2). Прогнать их через
`classifyResource`. Сюда же — макросы во внешних `.js`/`.css` (MAC-1/CSS-3/CJS-5): сканировать и их.

### C7 · Меньше regex по структурированному контенту 🟨 P1/P2
Проект верно ушёл на cheerio для HTML, но regex возвращается на краях и ломает/пропускает:
удаление `<script src>` в pipeline (**PIPE-2** → перенести в DOM), normalize-переписывание (NORM-4),
JS SW/eval (SW-1/EVAL-2 → ломают синтаксис), SVG (SVG-1/2 → XML-парсер). 

### C8 · Verify должен кликать 🟧 P1
**VR-1**: последний рубеж проверяет только пассивную загрузку → phone-home **по клику оффера** не
ловится → `ok=true` обманчив. Прокликивать CTA/квиз/формы (preventDefault), ждать таймеры; проверять
все страницы (VR-2). Плюс [CST-1](5b-verify-tools.md): тул должен отдавать путь к `clean-report.md`.

## Порядок работ

**P0 — дёшево + критично, сделать первым:**
1. **C1** — нормализация+схемы в `allowlist.ts` (закрывает AL-1/AL-2 и их проявления).
2. **C2** — снять «тихий SKIP» (PIPE-1/DOM-1).
3. **C3а** — siteDir-containment в normalize (NORM-1).
4. **REP-1** — починить отчёт (тип PHP + список удалений) — восстановить safety-net.

**P1 — высокий эффект, больше работы:**
5. **C4** — устойчивость детекторов к косвенности.
6. **C5б/в** — карантин-вместо-delete + корректность AST-хирургии (CJS-1/EUF-1).
7. **C3б** — `_external/` через allowlist (EXT-1).
8. **C8** — verify кликает (VR-1) + reportPath (CST-1).
9. **PIPE-2** — удаление ссылок в DOM, не regex.

**P2 — харднинг / реже встречается:**
10. **C6** — allowlist для preconnect/noscript/CSS-url; макросы во внешних файлах.
11. coverage-delete → флаг, не `rm` (COV-1/ANA-1); опц., default-off.
12. SVG-формы (SVG-1/2), CSS protocol-relative, regex-JS-проходы.
13. Изоляция ошибок (PIPE-3/WALK-1); артефакты вне деплой-каталога (QUAR-1); owner-config (POL-3).

## Что НЕ трогать (сильные стороны — сохранить)

- **Белый список default-deny** (`classifyResource`) — правильное ядро; чинить **вход** в него, не модель.
- **cheerio для HTML** — единственно верно (чинит битую вёрстку); проблема в краевых regex (C7), не в движке.
- **Карантин вместо тихого удаления** (HTML) + **бэкап по умолчанию** — распространить на JS (C5).
- **Репин известных либ на офиц. CDN + SRI (hard-путь)**, `fetchOfficial` хеширует **официальный** файл.
- **verify ловит запросы на чужие хосты** (реальный phone-home детектор) — добавить интерактив (C8).
- **Загрузчик base-aware** (резолвит+снимает `<base>`); coverage **блокирует** внешние запросы.
- **AST-скан строковых литералов для макросов** (анти-FP на `{a:1}`); **OWN_MACROS** как явный allowlist.

## Сквозные темы (одной строкой)

- **Блок-лист vs белый список:** граница — allowlist; каждый откат к блок-листу (preconnect/noscript/
  CSS/inline/JS-детекторы) — это FN-карман.
- **Литерал vs вычисляемое:** детекторы ждут литерал+прямой вызов, противник вычисляет/косвенит.
- **Warn vs действие:** redirect/keylogger/php/suspicious — только WARN → уезжают в прод, а отчёт их
  ещё и под-показывает (REP-1).
- **Delete vs карантин:** HTML карантинит (хорошо), JS/coverage удаляют (FP необратим).
- **Пассив vs интеракция:** verify/coverage гоняют пассивную загрузку → interaction-gated кража (клик
  по офферу — главный вектор арбитража) ускользает.
- **Regex vs парсер:** на краях regex возвращается и (а) ломает валидный код, (б) пропускает обход.
