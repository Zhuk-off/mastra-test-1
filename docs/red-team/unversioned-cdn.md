# Red Team: `utils/unversioned-cdn-detector.ts` (+ `detect-unversioned-lib.ts`, `registry/known-libs.ts`)

**Роль.** Подсистема репина библиотек **без версии в имени** (`jquery.min.js`, `vendor.js`).
Пре-скан `detectUnversionedLib` опознаёт либу по сигнатуре содержимого (первые 4КБ, реестр
`known-libs.ts`), затем `buildUnversionedCdnReplacements` строит замену на CDN + SRI. Включается
только в `--advanced`. Удалённые локальные файлы потом стирает `pipeline`.

---

## Находки

### [UCDN-1] 🟡 Robustness · 🟧 High · SRI считается от ЛОКАЛЬНОГО файла, а `src` → CDN

- **Сейчас:** `integrity = sha384(локальный файл)`, `cdnUrl = detected.lib.cdnUrl(version)`
  ([:40–44](src/mastra/cleaners/utils/unversioned-cdn-detector.ts:40)). Официальный файл **не
  фетчится** — хеш берётся с локальной копии.
- **Последствие:** браузер качает **CDN**-файл, считает его хеш, сравнивает с хешем **локального** →
  совпадение только при **байт-в-байт** идентичности. Локальные либы почти никогда не совпадают с CDN
  (другой минификатор, патч, пробелы, sourcemap-коммент) → **SRI mismatch → скрипт заблокирован →
  сайт ломается**. Хуже: `pipeline` затем **удаляет** локальный файл
  (`unversionedLibFilesToDelete`) → фолбэка нет → жёсткая поломка без восстановления.
  Контраст: hard-путь в `cdn-detector` хеширует **официальный** файл (правильно).
- **Рекомендация:** фетчить и хешировать **CDN**-файл (переиспользовать `fetchOfficial` из
  `cdn-detector`), либо не ставить SRI для unversioned-репина, либо не удалять локальный файл
  (оставить фолбэком).
- **Уверенность:** подтверждено чтением кода.

### [UCDN-2] 🟡 Robustness · 🟨 Medium · нет проверки существования CDN-URL

- **Сейчас:** в отличие от hard-пути, нет `fetchOfficial`-гейта — `cdnUrl` берётся на веру.
- **Последствие:** если `detected.lib.cdnUrl(version)` ведёт на 404 (несуществующая версия) →
  `src` указывает в 404 (+ SRI-mismatch из UCDN-1) → двойная поломка.
- **Рекомендация:** проверять существование CDN-URL перед заменой (как в hard-пути).
- **Уверенность:** подтверждено чтением кода.

### [UCDN-3] 🟢 Soundness · 🟨 Medium · версия из 4КБ + fallback → мисверсия минифицированных либ

- **Сейчас:** `detectUnversionedLib` смотрит только первые 4КБ
  ([:19](src/mastra/cleaners/passes/js-advanced/detectors/detect-unversioned-lib.ts:19)); если версия
  не найдена — `fallbackVersion` (хардкод в `known-libs.ts`, напр. jquery `3.7.1`).
- **Последствие:** у минифицированной либы со срезанным баннером версия в первые 4КБ не попадает →
  берётся `fallbackVersion` → репин на **чужую** версию. В связке с UCDN-1 это гарантированный
  SRI-mismatch. (Если и сигнатура не нашлась — либа просто не детектится, остаётся локальной — это
  fail-safe.)
- **Рекомендация:** искать версию во всём файле (не только 4КБ) или не репинить при неуверенной
  версии; fallback-версию использовать только без SRI.
- **Уверенность:** подтверждено чтением кода; зависит от наличия баннера в конкретной либе.

### [UCDN-4] 🟢 Soundness · 🟩 Low (безопасно, но громко) · FP по сигнатуре содержимого

- **Сценарий:** атакующий дописывает в `evil.js` баннер `/*! jQuery JavaScript Library v3.6.0 */`.
- **Сейчас:** `contentSignature` матчит баннер ([known-libs.ts:18](src/mastra/cleaners/registry/known-libs.ts:18))
  → файл опознан как jquery.
- **Последствие:** локальный `evil.js` **удаляется**, ссылка репинится на **чистый** CDN jquery
  (с SRI = хеш зловреда → mismatch → блок). То есть зловред **устранён** (заменён чистой либой) — это
  **fail-safe по безопасности**, но fail-loud по robustness (скрипт заблокирован). Не вектор «оставить
  зловред», а ещё один источник поломки.
- **Рекомендация:** отдельно — устранять через UCDN-1; FP сам по себе не опасен.
- **Уверенность:** подтверждено чтением кода.

## Заметки

- `KNOWN_LIBS` — всего 5 (jquery/bootstrap/popper/swiper/lodash). Остальное не детектится (FN) →
  остаётся локальным → разбирается JS-чистильщиком (Тир 3). Это приемлемый fail-safe, не дыра.
- Реестр сигнатур — удобство, не граница безопасности.

## Пробелы в тестах

Не нашёл прямых тестов на эту подсистему. Нужны: UCDN-1 (SRI от локали vs CDN), UCDN-2 (404 CDN-URL),
UCDN-3 (fallback-версия для минифицированной либы), и интеграционный «локальная либа ≠ CDN → mismatch».

## Итог

1. **UCDN-1** — High: хешировать CDN-файл, а не локальный (иначе репин unversioned-либ системно
   ломает сайт). Это **главный** фикс группы по robustness.
2. **UCDN-2/UCDN-3** — проверка существования и корректная версия.

## ✅ Статус фиксов

- **UCDN-1 ✅ + UCDN-2 ✅** — `buildUnversionedCdnReplacements` больше НЕ хеширует локальный файл.
  Теперь зовётся `fetchOfficial(cdnUrl)` (экспортирован из `cdn-detector.ts`): SRI считается от
  ОФИЦИАЛЬНОГО CDN-файла (UCDN-1 — браузер сверяет именно его), и если CDN-URL недоступен (404/сеть,
  `ok=false`) — замена НЕ добавляется (UCDN-2), значит локальный файл не удаляется и остаётся фолбэком.
  Унифицировано с hard-путём `cdn-detector` (тот же `fetchOfficial`: sha384-стрим, лимит 5МБ, таймаут
  4с, LRU-кэш). Тесты: новый `unversioned-cdn-detector.test.ts` (мок fetch: SRI=хеш CDN, не локали; 404→пропуск).
- **UCDN-3 / UCDN-4** — НЕ трогали (версия из 4КБ+fallback; FP по сигнатуре). UCDN-3 частично смягчён:
  при мисверсии CDN-URL вероятнее 404 → теперь это пропуск (фолбэк), а не битый SRI. Остаются 🆕.
