# Red Team — security-аудит чистильщика лендингов

Этот каталог — **аудит модели угроз** системы очистки лендингов (`src/mastra/cleaners/**`):
систематический разбор, где злоумышленник протащит кражу трафика / exfil / редирект мимо
очистки, и где легитимный (но необычный) лендинг сломается. Это **reference и бэклог**,
а не список живых багов.

## Статус

Аудит проведён V–VI 2026; фиксы прошли в ветке `redteam-fixes` и **смерджены в `main`**
(PR #1). Закрыто всё **Critical + High** и основная масса значимых **Medium**; остаток
(🆕 ниже) — точечная робастность/соундность без блокеров.

Легенда колонки **Status** в [`_index.md`](_index.md):

- ✅ — исправлено, код в `main`, закреплено тестами.
- 🛠 — закрыто частично / по основному пути; остаток описан в самой находке.
- 🆕 — **открытый бэклог**: проанализировано, фикс не делали. Это НЕ регрессия — осознанно отложено.
- 🟩 — мелочь/гигиена, низкий приоритет.
- 🚫 — won't-fix: осознанное решение владельца (напр. OFFER-1 — агрессивная offer-политика
  by-design; QUAR-1 — артефакты внутри папки карантина).

## Навигация

- **[`00-summary.md`](00-summary.md)** — начни отсюда: корневые кластеры (мало фиксов →
  много находок) и приоритетный порядок работ.
- **[`_index.md`](_index.md)** — полный реестр находок: severity, статус, методология
  («три угла»: Bypass / Robustness / Soundness), формат находки.
- Остальные файлы — детальные разборы по кластерам (`2a`…`5b`) и по модулям
  (`pipeline.md`, `normalize-landing-structure.md`, `html-dom.md`, `cdn-detector.md`, …).

## Замечания

- Доки фактически описывают **как обойти чистильщик** (path traversal, exfil-байпасы и т.п.).
  Для приватного репозитория это нормально; перед публикацией стоит пересмотреть.
- Открытые 🆕-пункты при желании выносятся в issue-трекер для actionable-формы; для соло-работы
  `_index.md` сам по себе служит лёгким бэклогом.
